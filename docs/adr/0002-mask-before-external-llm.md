# ADR-0002: 외부 LLM 호출 전 마스킹 (fail-closed)

- 상태: Accepted
- 날짜: 2026-06-20
- 관련: [docs/06-security-privacy.md](../06-security-privacy.md), [MEMORY.md](../../MEMORY.md)

## 맥락

입주민 문의·문서에는 개인정보(이름·동호수·연락처·차량번호 등)가 섞인다. 응대·요약을 위해
외부 LLM(Claude 등)을 호출하는데, 개인정보가 외부로 유출되면 법규 위반·신뢰 붕괴다.

## 결정

외부 LLM API 호출 **직전** 개인정보를 마스킹/가명화한다. 마스킹이 실패하면
**호출을 중단(fail-closed)**한다 — "일단 보내고 나중에 처리"는 없다.

## 대안

- **호출 후 로그에서 제거**: 이미 외부로 나감. 무의미. 기각.
- **fail-open(마스킹 실패해도 호출)**: 가용성은 좋으나 유출 위험. 개인정보 앞에서 가용성 < 안전. 기각.
- **모델 온프레미스**: 비용·품질 트레이드오프. 현 단계 범위 밖(재검토 가능).

## 결과

- 마스킹 레이어는 LLM 어댑터 경계에 강제 삽입. 우회 경로 없어야 함.
- 마스킹 실패는 에러가 아니라 **정상적 차단 경로** — 담당자 연결 폴백으로 이어진다.
- 테스트에서 마스킹 우회는 CRITICAL 게이트로 취급([docs/07](../07-testing-strategy.md)).
- 재검토 신호: 검증된 온프레미스/프라이빗 LLM 도입 시.
