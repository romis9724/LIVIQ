# ADR-0006: 관리비 원천 = 관리자 엑셀 업로드 (ERP 어댑터는 추후)

- 상태: Accepted (ADR-0003 대체)
- 날짜: 2026-07-13
- 관련: [ADR-0003](0003-erp-single-source-fees.md), [docs/03 §4.6](../03-database-design.md), [docs/11 §3.3](../11-data-architecture.md)

## 맥락

ADR-0003은 "관리비는 ERP가 단일 출처(읽기 전용 미러)"였다. 그러나 대상 단지에
연동 가능한 ERP가 없고, 관리소는 월관리비를 엑셀로 관리한다. 90세대 소규모라
수기 업로드가 현실적이다.

## 결정

**관리자 엑셀 업로드가 관리비의 단일 출처**다. 업로드 → 파싱·Zod 검증 → 오류 리포트 →
미리보기 → 확정 시 해당 월 `fees` 전체 교체. 컬럼은 동·호·부과월 + 항목별 금액.
AI는 여전히 **설명만** 한다(계산·부과 금지 — 0003의 이 원칙은 유지).
ERP가 생기면 `source` 컬럼(excel|erp)과 어댑터 인터페이스로 병행한다.

## 대안

- **ERP 미러 유지(0003)**: 연동할 ERP가 없음. 기각.
- **화면에서 세대별 수기 입력**: 90세대×항목 수를 매월 입력 — 엑셀 업로드보다 느리고 오류 유발. 기각.
- **AI가 엑셀 해석·자동 반영**: 금액 데이터에 LLM 개입은 규칙(계산 금지) 위반. 파싱은 결정적 코드로. 기각.

## 결과

- `fees`·`excel_uploads` 테이블 신설, 업로드 권한 MANAGER/STAFF 한정 + 감사 로그.
- 오업로드 위협은 검증·미리보기·확정 2단계와 이력으로 완화([docs/06 §1](../06-security-privacy.md)).
- 재검토 신호: ERP 도입 계약 시(어댑터 추가, 원천 전환은 새 ADR).
