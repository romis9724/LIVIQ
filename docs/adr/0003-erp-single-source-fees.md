# ADR-0003: 관리비는 ERP 단일 출처, AI는 설명만

- 상태: Accepted
- 날짜: 2026-06-20
- 관련: [docs/00-requirements.md](../00-requirements.md), [MEMORY.md](../../MEMORY.md)

## 맥락

입주민이 관리비를 문의하면 AI가 답할 수 있어야 한다. 그러나 관리비는 법적·회계적으로
민감하며 이미 ERP(관리 시스템)가 계산·부과의 공식 주체다. AI가 값을 계산하면
불일치·오류·책임 소재 문제가 생긴다.

## 결정

관리비의 **계산·부과는 전적으로 ERP**가 담당한다. AI는 ERP가 산출한 값을 **조회·설명**만 한다.
AI가 금액을 재계산하거나 새로 부과하지 않는다.

## 대안

- **AI가 명세 기반 재계산**: ERP와 미세 불일치 → 신뢰 붕괴·분쟁. 기각.
- **AI가 부과까지 자동화**: 부수효과를 LLM 출력이 트리거 → 절대규칙 8 위반. 기각.

## 결과

- 관리비 화면(`web-resident/fees`)은 ERP 값 표시 전용. 계산 로직 없음.
- AI 답변은 "이 항목은 …로 산정되었습니다(출처: ERP/규약 조항)" 형태의 설명.
- 재검토 신호: ERP가 설명 API를 제공하지 못해 AI 보조 계산이 불가피할 때(그래도 표시는 ERP 값).
