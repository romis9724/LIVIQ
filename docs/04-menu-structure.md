> 역할 코드는 [00 §2](00-requirements.md) 기준. 메뉴는 **역할별로 노출이 다르다**(RBAC).

## 1. 앱 분리

| 앱 | 대상 | 형태 |
|----|------|------|

## 2. 입주민 웹 (모바일 우선)

**진입 온보딩**: 로그인 화면 **회원가입 버튼** → 단지 선택(공개 단지 목록 — 단지별 가입 링크 `?t=`는 사전 선택 딥링크) → 이메일+비밀번호 가입 → **이메일 검증 메일** 확인(검증 전 로그인 불가) → 로그인 → **개인정보 동의**(목적별 필수/선택) → 성함·생년월일·동·호수 입력(**만 14세 미만 차단**) → 승인 대기 화면(소장 승인 전 기능 차단) → 승인 후 아래 탭 진입. 비밀번호 재설정('비밀번호 찾기')도 제공.

하단 탭 5개 + 비서 중심.

```text
입주민 앱
├── 🏠 홈
│   ├── 우리 단지 요약 (공지 N건, 처리중 민원, 이번달 관리비)
│   └── 바로가기 (비서 질문, 민원접수, 관리비)
├── 💬 AI 비서 (핵심)
│   ├── 자연어 질의 (스트리밍 응답)
│   ├── 응답 + 출처 카드(원문 링크) + 👍/👎
│   ├── 추천 질문 (주차/공사시간/반려동물 규정 …)
│   └── "담당자 연결" (신뢰도 낮음/요청 시)
├── 📢 공지
│   ├── 공지 목록 (상단 고정·말머리 배지)
│   └── 공지 상세 (첨부 다운로드)
├── 📝 민원·하자
│   ├── 접수 (사진 첨부, AI 카테고리 추천)
│   ├── 내 민원 목록 + 처리상태
│   └── 민원 상세 (타임라인)
├── 💰 관리비
│   ├── 이번달/추이 (그래프)
│   ├── 항목 상세 (확정 관리비 데이터)
│   └── "왜 올랐나요?" → AI 설명 (전월·평균 대비)
└── 👤 나
    ├── 내 활동 (질의·민원·피드백 이력)
    ├── 우리집 평면도 (본인 세대만, 장치 마커·팝업)
    ├── 알림함 (인앱, 읽음 배지)            ← 웹푸시 Phase 2
    ├── 세대/알림 설정
    ├── 개인정보 동의 관리
    └── 회원 탈퇴 / 로그아웃
```

> 검색형 답변은 비서에 일원화. 공지/민원/관리비 화면은 "조회"에 집중하고, 의문이 생기면 비서로 자연스럽게 유도(딥링크).

## 3. 관리자 웹 (데스크톱 우선)

좌측 사이드바 네비. **역할별 메뉴 가시성**을 우측에 표기.

> 모바일(≤768px)에서도 **조회 + 승인 액션**은 보장하고, 넓은 테이블은 카드 레이아웃으로 폴백한다.

```text
관리자 콘솔
├── 📊 대시보드            [MANAGER]              자동해결률·환각률·토큰비용·민원현황
├── 🙋 주민 관리           [MANAGER] (H7-9 — 가입 승인에서 개편, 목록 위주)
│   ├── 요약 통계 (명부 총계·미가입·가입완료·전출후보·승인 대기)
│   ├── 가입 대기 (자동 대조 — 일치 배지·불일치 사유)
│   ├── 명부 목록 (동·호·성함 마스킹·상태, 검색·필터·페이지네이션)
│   └── 명부 엑셀 업로드 (접이식 + 마지막 업로드 일시·건수·오류 상시)
├── 📝 민원 관리           [MANAGER STAFF]
│   ├── 접수함 (AI 분류·우선순위)
│   ├── 배정/처리
│   └── 처리 통계
├── 📢 공지사항             [MANAGER STAFF]
│   ├── 공지 목록 (고정·상태 배지: 임시저장/예약/발행)
│   ├── 작성 (제목·본문·분류(NOTICE_CATEGORY 코드 선택)·행사/작업 기간·대상 동·키워드·대상·첨부 업로드)
│   ├── 수정 / 삭제 · 상단 고정
│   └── 임시저장 / 예약 발행 (예약 시각 도달 시 자동 발행)
├── ✅ AI 검수 큐          [MANAGER]              신뢰도 낮은 AI 응답 검토·승인/수정
├── 📄 문서 관리           [MANAGER STAFF]
│   ├── 업로드 (PDF/HWP/이미지 · 카테고리(DOC_CATEGORY 코드 선택, 필수))
│   ├── 공개범위 설정 (ALL/RESIDENT/ADMIN)
│   ├── 문서 검색 (자연어 → 원문 발췌·링크)
│   └── 색인 상태 (pending/indexed/failed, 재색인)
├── 💰 관리비 관리          [MANAGER]
│   ├── 엑셀 업로드 (검증·오류 리포트·미리보기·확정)   ← 같은 월 재업로드=전체 교체
│   ├── 월별 부과 현황
│   └── 세대별 조회
├── 🔧 시설 관리           [MANAGER]
│   ├── 설비 상태 목록 (정상/점검/장애/위험)
│   ├── 유지보수·장애 이력
│   └── 시설 AI 도우미 (유사 장애 검색→원인 후보)
├── 🧊 단지 트윈           [MANAGER] (H9 — geometry 있는 단지만 노출(`/me`의 `has_twin`), [ADR-0019](adr/0019-complex-twin-3d.md))
│   ├── 3D 단지 뷰 (deck.gl — 세대 폴리곤, 클릭=세대 상세)
│   ├── 상태 오버레이 토글 (입주·민원·관리비·설비(동 단위))
│   └── 세대 상세 패널 (세대원 명부 마스킹 표시·미종결 민원·당월 관리비)
│                        · geometry 업로드(units.json)는 설정>동/호수 관리에 배치
├── 🏢 단지 관리           [MANAGER]
│   ├── 동·세대 마스터 (101~103동·15층·90세대)
│   ├── 평면도 타입 (배경 이미지 업로드, 기본 장치 좌표)
│   ├── 장치 좌표 에디터 (타입 기본 + 세대 오버라이드)
│   └── 단지 배치도·공용층 (시설 포인트)
├── 👥 직원 관리           [MANAGER] (STAFF 초대 링크 메일·비활성화·삭제, 타 소장 삭제 — H7-5 최상위 승격·H7-6 수명주기)
└── ⚙️ 설정               [MANAGER] (H8-4·H8-5 — [ADR-0017](adr/0017-tenant-code-registry.md))
    ├── ⚙️ 코드 관리         [MANAGER] (H8-4 — 그룹 선택 → 코드 트리: 추가·수정·정렬·비활성·삭제, is_system 그룹 잠금)
    ├── 🏠 동/호수 관리      [MANAGER] (H8-5 — 동 CRUD·세대 목록·층/호 범위 일괄 생성·개별 삭제)
    │                        · 삭제 보호: 입주민·명부·민원·관리비 연결 세대/세대 보유 동은 삭제 불가(409)
    │                        · 초기 대량 등록은 seed_households_xlsx.py 스크립트 유지
    │                        · 트윈 geometry 업로드(units.json — 검증 리포트·전체 교체, H9)
    └── (Phase 2 — 민원 카테고리·SLA, ERP 연동 상태, 개인정보/보관정책)
```

> **SYS_ADMIN 뷰(web-admin 내)**: SYS_ADMIN 로그인 시 위 소장 메뉴 대신 **단지 관리** 전용 뷰만 노출 — 단지 생성·소장 초대(단지당 1명)·소장 제거·빈 단지 삭제·단지 비활성화/재활성화(단지 콘텐츠 비열람, H7-6).
> **Phase 2**: 입대의·선관위 전용 게시판·안건은 Phase 2. MVP의 회의록은 문서 관리(DOC_CATEGORY 코드 "회의록")로 업로드·열람.

> ¹ 공지사항은 H8-1에서 게시판으로 전환 — STAFF도 **작성·발행** 가능(검수·관리비·시설은 소장 전용 유지, [ADR-0015](adr/0015-notice-board-replaces-ai-draft.md)).
> ² 설정 메뉴는 MANAGER 전용. STAFF는 설정 메뉴 미노출이나, 공지·문서 작성 폼이 분류 코드를 채우기 위해 `GET /admin/code-groups`(조회 전용)를 소비한다([01 §13](01-architecture.md), H8-4).
> 동/호수 관리(H8-5)는 기존 🏢 단지 관리의 동·세대 마스터를 설정 하위 CRUD로 정식화한다(중복 아님 — 단지 관리는 평면도·좌표 중심, 설정은 동·세대 데이터 CRUD).
> 가시성은 프론트 숨김 + **API 인가(서버)** 이중 적용([06]). 프론트 숨김만으로 보호하지 않는다.

## 5. 핵심 사용자 여정 (Top Tasks)

| 사용자 | 과업 | 경로 |
|--------|------|------|
| SYS_ADMIN | 단지 개설·소장 초대 | 단지 관리 → 단지 생성 → 소장 이메일 초대(링크 메일) |
| 관리소장 | 직원 초대 | 직원 관리 → 직원 이메일 초대(링크 메일) → 직원이 링크로 비밀번호 설정 |
| 입주민 | 이메일 가입 | 가입(이메일·비밀번호) → 검증 메일 확인 → 로그인 → 동의·동호 입력 → 대기 → 소장 승인 |
| 입주민 | "공사 가능 시간?" | 비서 → 질의 → 출처 인용 답변 |
| 입주민 | 누수 신고 | 민원 → 접수(사진) → 상태 추적 |
| 입주민 | 관리비 급증 이유 | 관리비 → "왜 올랐나요?" → AI 설명 |
| 입주민 | 우리집 공유기 위치 확인 | 나 → 우리집 평면도 → 마커 클릭 → 팝업 |
| 관리소장 | 신규 가입 승인 | 주민 관리 → 가입 대기 → 명부 대조(일치 배지·불일치 사유) → 승인/거절(사유) |
| 관리소장 | 단수 공지 | 공지사항 → 작성(+첨부) → 발행/예약 |
| 관리소장 | 월관리비 등록 | 관리비 관리 → 엑셀 업로드 → 검증·미리보기 → 확정 |
| 관리소장 | 펌프 이상 원인 | 시설 → AI 도우미 → 유사 장애·이력 |
| 관리소장 | 단지 현황 한눈에 | 단지 트윈 → 오버레이(민원/관리비/입주) → 세대 클릭 → 상세 패널 |
| 관리직원 | 민원 처리 | 민원 → 접수함(AI 분류) → 배정 → 완료 |

## 6. 공통 IA 규칙

- URL = 상태: 필터·탭·검색은 search params로 ([../README](../README.md) web 패턴).
- 빈/로딩/오류/권한없음 상태를 모든 목록 화면에서 명시 설계([05]).
- 네비게이션 깊이 ≤ 3. 핵심 과업은 2탭 이내.
- **화면-엔드포인트 매핑·인가 역할은 [01 §13 REST API 표면](01-architecture.md)이 소유**한다(여기 중복 기술 금지).
