# 04. 메뉴 구조 / 정보 구조(IA)

> 요구사항: [00-requirements.md](00-requirements.md) · UI/UX: [05-ui-ux-design.md](05-ui-ux-design.md)
> 역할 코드는 [00 §2](00-requirements.md) 기준. 메뉴는 **역할별로 노출이 다르다**(RBAC).

## 1. 앱 분리

| 앱 | 대상 | 형태 |
|----|------|------|
| `web-resident` | 입주민(RESIDENT) | 반응형 웹/PWA, 모바일 우선 |
| `web-admin` | MANAGER/STAFF/FACILITY/COUNCIL | 데스크톱 우선 반응형 |
| (운영) | SYS_ADMIN | 별도 최소 콘솔(모니터링·테넌트), 단지 콘텐츠 비열람 |

## 2. 입주민 웹 (모바일 우선)

**진입 온보딩**: Google 로그인 → (최초) **단지 초대코드 입력**(tenant 확정) → **개인정보 동의**(목적별 필수/선택) → 성함·생년월일·동·호수 입력(**만 14세 미만 차단**) → 승인 대기 화면(소장 승인 전 기능 차단) → 승인 후 아래 탭 진입.

하단 탭 5개 + 비서 중심.

```text
입주민 앱
├── 🏠 홈
│   ├── 우리 단지 요약 (공지 N건, 처리중 민원, 이번달 관리비)
│   └── 바로가기 (비서 질문, 민원접수, 관리비)
├── 💬 AI 비서 (핵심)
│   ├── 자연어 질의 (스트리밍 응답)
│   ├── 응답 + 출처 카드(원문 링크) + 👍/👎
│   ├── 추천 질문 (주차/공사시간/반려동물 규정 …)
│   └── "담당자 연결" (신뢰도 낮음/요청 시)
├── 📢 공지
│   ├── 공지 목록 (말머리·중요 배지)
│   └── 공지 상세
├── 📝 민원·하자
│   ├── 접수 (사진 첨부, AI 카테고리 추천)
│   ├── 내 민원 목록 + 처리상태
│   └── 민원 상세 (타임라인)
├── 💰 관리비
│   ├── 이번달/추이 (그래프)
│   ├── 항목 상세 (확정 관리비 데이터)
│   └── "왜 올랐나요?" → AI 설명 (전월·평균 대비)
└── 👤 나
    ├── 내 활동 (질의·민원·피드백 이력)
    ├── 우리집 평면도 (본인 세대만, 장치 마커·팝업)
    ├── 알림함 (인앱, 읽음 배지)            ← 웹푸시 Phase 2
    ├── 세대/알림 설정
    ├── 개인정보 동의 관리
    └── 회원 탈퇴 / 로그아웃
```

> 검색형 답변은 비서에 일원화. 공지/민원/관리비 화면은 "조회"에 집중하고, 의문이 생기면 비서로 자연스럽게 유도(딥링크).

## 3. 관리자 웹 (데스크톱 우선)

좌측 사이드바 네비. **역할별 메뉴 가시성**을 우측에 표기.

> 모바일(≤768px)에서도 **조회 + 승인 액션**은 보장하고, 넓은 테이블은 카드 레이아웃으로 폴백한다.

```text
관리자 콘솔
├── 📊 대시보드            [MANAGER]              자동해결률·환각률·토큰비용·민원현황
├── 🙋 가입 승인           [MANAGER]
│   ├── 명부 엑셀 업로드 (사전등록: 성함·생년월일·동·호)
│   ├── 대기 목록 (자동 대조 결과·"명부 일치" 배지)
│   └── 승인/거절(사유)
├── 📝 민원 관리           [MANAGER STAFF]
│   ├── 접수함 (AI 분류·우선순위)
│   ├── 배정/처리
│   └── 처리 통계
├── 📢 공지 관리           [MANAGER STAFF]
│   ├── 공지 목록
│   ├── 초안 작성 (키워드→AI 초안)        ← 자동발송 금지
│   └── 발송(검수 후) / 예약
├── ✅ AI 검수 큐          [MANAGER]              신뢰도 낮은 AI 응답 검토·승인/수정
├── 📄 문서 관리           [MANAGER STAFF]
│   ├── 업로드 (PDF/HWP/이미지)
│   ├── 공개범위 설정 (ALL/RESIDENT/ADMIN/COUNCIL)
│   ├── 문서 검색 (자연어 → 원문 발췌·링크)
│   └── 색인 상태 (pending/indexed/failed, 재색인)
├── 💰 관리비 관리          [MANAGER STAFF]
│   ├── 엑셀 업로드 (검증·오류 리포트·미리보기·확정)   ← 같은 월 재업로드=전체 교체
│   ├── 월별 부과 현황
│   └── 세대별 조회
├── 🔧 시설 관리           [FACILITY MANAGER]
│   ├── 설비 상태 목록 (정상/점검/장애/위험)
│   ├── 유지보수·장애 이력
│   └── 시설 AI 도우미 (유사 장애 검색→원인 후보)
├── 🗳️ 입대의·선관위        [COUNCIL MANAGER]
│   ├── 전용 게시판 (Phase 2 — MVP는 문서 열람만)
│   └── 회의록 문서 (문서 관리로 업로드·열람, source_type=회의록)
├── 🏢 단지 관리           [MANAGER]
│   ├── 동·세대 마스터 (101~103동·15층·90세대)
│   ├── 평면도 타입 (배경 이미지 업로드, 기본 장치 좌표)
│   ├── 장치 좌표 에디터 (타입 기본 + 세대 오버라이드)
│   └── 단지 배치도·공용층 (시설 포인트)
└── ⚙️ 설정               [MANAGER]
    ├── 사용자·역할 (역할 부여는 MANAGER; 첫 계정은 테넌트 시드 초대)
    ├── 민원 카테고리·SLA
    ├── ERP 연동 상태 (추후)
    └── 개인정보/보관정책
```

## 5. 핵심 사용자 여정 (Top Tasks)

| 사용자 | 과업 | 경로 |
|--------|------|------|
| 입주민 | "공사 가능 시간?" | 비서 → 질의 → 출처 인용 답변 |
| 입주민 | 누수 신고 | 민원 → 접수(사진) → 상태 추적 |
| 입주민 | 관리비 급증 이유 | 관리비 → "왜 올랐나요?" → AI 설명 |
| 입주민 | 우리집 공유기 위치 확인 | 나 → 우리집 평면도 → 마커 클릭 → 팝업 |
| 관리소장 | 신규 가입 승인 | 가입 승인 → 대기 목록 → 명부 대조 → 승인/거절(사유) |
| 관리소장 | 단수 공지 | 공지 → 초안(키워드) → 검수 → 발송 |
| 관리소장 | 월관리비 등록 | 관리비 관리 → 엑셀 업로드 → 검증·미리보기 → 확정 |
| 관리직원 | 민원 처리 | 민원 → 접수함(AI 분류) → 배정 → 완료 |
| 시설담당 | 펌프 이상 원인 | 시설 → AI 도우미 → 유사 장애·이력 |
| 입대의 | 회의록 등록·열람 | 문서 관리 → 업로드(회의록) → 색인 → 검색·열람 |

## 6. 공통 IA 규칙

- URL = 상태: 필터·탭·검색은 search params로 ([../README](../README.md) web 패턴).
- 빈/로딩/오류/권한없음 상태를 모든 목록 화면에서 명시 설계([05]).
- 네비게이션 깊이 ≤ 3. 핵심 과업은 2탭 이내.
- **화면-엔드포인트 매핑·인가 역할은 [01 §13 REST API 표면](01-architecture.md)이 소유**한다(여기 중복 기술 금지).
