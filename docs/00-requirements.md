# 00. 요구사항 정의서 (SRS)

> 상위 기획: [../README.md](../README.md) · 문서 인덱스: [README.md](README.md)
> 본 문서는 LIVIQ가 **무엇을** 해야 하는지를 정의한다. **어떻게**는 [01-architecture.md](01-architecture.md) 이하에서 다룬다.

## 1. 목적과 범위

LIVIQ는 기존 아파트 관리 시스템·문서 위에 얹는 **AI 검색·응대·요약 계층**이다.
MVP는 **입주민 반응형 웹/PWA + 관리자 웹 + AI 코어**를 동시에 구축한다.

> 제품 경계: **파일럿 최소 운영앱 + AI 계층** — 기존 상용앱 대체가 아니라 파일럿 단지 운영에 필요한 최소 화면 + AI 차별화(결정 기록).

- **In Scope (MVP)**: 문서 RAG 검색(회의록은 문서로 업로드·검색), 민원 1차 AI 응대, 공지사항 게시판(첨부파일 포함), 관리비 설명, 시설 이력 검색, 입주민 온보딩·가입 승인(명부 대조), 2D 세대 평면도(정적), 월관리비 엑셀 업로드·조회, 입주민/관리자 기본 운영 화면
- **Out of Scope (MVP)**: 회계 부과 계산, 결제/수납 처리, 회의 음성 STT·회의록 자동 요약(추후), 3D 디지털 트윈/BIM, 재난 대피 경로(9장 R&D), 네이티브 앱
- **비목표**는 [../README.md](../README.md) §1.4를 따른다.

## 2. 사용자 및 역할 (RBAC 기준)

| 역할 | 코드 | 설명 | 주요 권한 |
|------|------|------|-----------|
| 입주민 | `RESIDENT` | 이메일+비밀번호 자체 인증 + 소장 승인 후 활성 세대 구성원 | 본인 세대 정보, 공개 문서 검색, 민원 등록, AI 질의 |
| 관리소장 | `MANAGER` | 관리사무소 책임자 | 단지 전체 운영, 공지 발송, 민원 배정, 검수, **직원 초대** |
| 관리직원 | `STAFF` | 관리사무소 직원 | 민원 처리, 문서 업로드, 공지사항 작성·발행 (검수·관리비·시설은 소장 전용 — H8-1 개정) |
| 시스템관리자 | `SYS_ADMIN` | LIVIQ 운영자 | **단지 생성·소장 초대**, 모니터링, 골든셋 관리 (단지 데이터 직접 열람 불가, §보안) |

> 멀티테넌시: `RESIDENT`·`MANAGER`·`STAFF`는 특정 **단지(tenant)** 에 귀속된다. `SYS_ADMIN`은 **시스템 테넌트**(고정 UUID) 소속으로 단지 생성·소장 초대만 수행(단지 콘텐츠 비열람).
> **Phase 2 역할**: `FACILITY`(설비 유지보수)·`COUNCIL`(입대의·선관위)은 파일럿 범위 제외 — 시설 기능은 MANAGER 전용 운영, 전용 게시판·안건은 Phase 2 재도입 여지.

| FR-ONB-12 | **관리 계정·단지 수명주기(H7-6)**: 직원·소장 **삭제 = 소프트 삭제+PII 비식별**(세션 revoke, 복구 불가, 동일 이메일 재초대 가능 — 삭제 주체: 직원←소장, 소장←다른 소장·SYS_ADMIN, 자기 자신 불가). 비활성화(복구 가능)는 별도 유지. **단지 삭제는 빈 단지만**(주민·콘텐츠 존재 시 거부), 운영 중 단지는 **비활성화**(소속 로그인 차단·가입 목록 제외·세션 revoke, 재활성화 가능) | M | 상 |

> 명부는 엑셀 일괄 업로드로 사전등록되며(가입 전 개인정보 — [06](06-security-privacy.md) 법적 근거·보관 참조), 가입 시 입력값(성함·생일·동·호) 자동 대조로 "명부 일치"를 표시한다(불일치는 소장 수동 판단, 자동 승인 없음). 승인/거절/비활성화는 감사 로그 기록.

## 5. 제약 (Constraints)

- **법규**: 공동주택관리법(규약·회의록·관리비 공개 의무, AI는 공식 고지 대체 불가), 개인정보보호법(동의·목적제한·보관), 소방시설법(대피 안내는 인증 설비 책임 — MVP 제외).
- **데이터**: LLM에 개인정보 미전송(전 프로바이더, self-hosted 포함). 외부 API 사용 시 학습 비사용·미보관 계약 필수.
- **단일 출처**: 관리비 단일 출처는 확정 업로드 데이터(현재 엑셀, 추후 ERP). AI는 설명만, 계산·부과 금지.
- **사람 검수**: 위험도 높은 AI 출력은 검수 게이트 통과 필수. 공지사항은 H8-1에서 관리사무소가 직접 작성하는 게시판으로 전환되어 AI가 관여하지 않는다([ADR-0015](adr/0015-notice-board-replaces-ai-draft.md)) — "AI 자동발송 금지" 원칙은 assistant 등 다른 AI 표면에 유지.

## 6. 가정 및 의존성

- 파일럿 단지 1곳 이상, 세대 정보·문서 제공.
- `user`↔`household`는 **1:1**. 단지 내 이사는 소장이 `household_id`를 변경(캐시 무효화 연동). 한 계정의 **다세대(복수 세대) 소유는 파일럿 미지원**(2026-07-13 확정).
- 기존 ERP 연동은 없음(추후 병행 여지). 관리비는 관리자 엑셀 업로드로 확보.
- 관리규약 등 핵심 문서가 전자문서로 확보 가능(아니면 OCR 비용 발생).
- 평면도 원본은 이미지/PDF 스캔으로 확보(CAD 벡터 도면 없음) → 배경 이미지 + 좌표 레이어 방식.
- LLM은 OpenAI-호환 엔드포인트(Ollama·vLLM·OpenAI 등) 중 **단일 모델** 운영, 승인된 인프라에서 실행. 모델 선정·교체는 골든셋 비교([07](07-testing-strategy.md)).
- 파일럿 기본 LLM = **Ollama 로컬**(생성 + 임베딩 bge-m3). 품질 미달 시 골든셋 비교로 외부 API 승격(교체는 env만) — [ADR-0005](adr/0005-single-llm-openai-compat.md) 갱신 이력.

## 7. 성공 지표 (KPI)

수용 기준으로 삼는 KPI. 목표는 검증 전 **가정값**이며 파일럿 데이터로 보정한다.

| 지표 | 목표(가정) |
|------|-----------|
| 문의 1차 자동해결률 | 40% |
| 응답 만족도 | 70%+ |
| 문서 검색 시간 | 50% ↓ |
| ~~공지 작성 시간 60%↓~~ | H8-1 폐기 — AI 초안 제거로 무효([ADR-0015](adr/0015-notice-board-replaces-ai-draft.md)) |
| 환각률 | ≤ 5% |

> go/no-go 판정 계약(측정 기간·표본 수·판정자)은 파일럿 시작 시 확정한다.
