## 1. 종합 판정

보안·관리비 SSOT·읽기전용 에이전트 축(`00`–`03`·`06`·ADR-0005/0006/0007)은 문서군 중 가장 잘 정렬되어 있다.

다만 아래가 구현 착수 전 해소 대상이다.

- ADR 번호 이원화 (`01` §12 vs `docs/adr/`)
- `10`·핸드오프·`MEMORY`의 stale (Claude / ERP)
- “AI 계층” 정체성 vs `00`/`04` 풀앱 MVP 범위
- 검수 큐·KPI·Neo4j 테스트 게이트 공백

## 2. 강점

1. **절대 규칙 ↔ 핵심 설계 정합** — 출처 강제·PII fail-closed·tenant+RLS·관리비 설명만·공지 자동발송 금지·액션=코드가 `00`/`01`/`03`/`06`과 ADR에 반복 고정됨.
2. **관리비 원천 전환 추적성** — ADR-0003 Superseded → ADR-0006 Accepted. `00` FR-FEE·`03` §4.6·`11` §3.3이 엑셀 SSOT로 정렬.
3. **에이전트 가드레일** — ADR-0007 / `01` §5.2: 읽기 전용 + 스텝 상한 + 코드 검증. 자유 ReAct 배제.
4. **데이터 SoR** — PG SoR · Neo4j 파생 · 임베딩 중복 금지 · outbox 원자 기록 (`03`·`11`).
5. **AI 신뢰 UX (`05`)** — 출처 카드·검수·자동발송 금지가 정체성과 맞음. (문제는 IA `04` 범위 폭.)

## 3. Critical 이슈

### ~~C1. ADR 번호 이원화~~ ✅ 반영(2026-07-13)

> 반영 — `01` §12·`09` §10을 `docs/adr/` 정본 링크 표로 교체

- **위치**: `01` §12 ADR-1~9 요약표 vs `docs/adr/0001`–`0007` vs `09` §10
- **예시**: §12 “ADR-2 = 단일 LLM” ≠ 파일 `0005`; §12 “ADR-8 엑셀” ≈ adr `0006`
- **왜 문제**: Accepted 결정의 SSOT가 둘이라 변경·supersede 추적이 깨짐
- **제안**: `docs/adr/`만 정본. `01` §12·`09` §10은 링크 표로 교체

### ~~C2. `10-project-plan`·핸드오프가 구버전 SSOT~~ ✅ 반영(2026-07-13)

> 반영 — `10` 역사적 배너·핸드오프 카피 교체

- **위치**: `10` §8 (Claude/GPT·하이브리드·ERP 주경로), `design-handoff-prompt.md` (출처=ERP)
- **정본**: ADR-0005 (OpenAI-호환 단일), ADR-0006 (엑셀 업로드)
- **왜 문제**: `docs/README`가 10을 “구축 전제 실행 계획”으로 안내 → 착수 시 잘못된 시스템 구축
- **제안**: 10을 ADR·`00`에 맞게 전면 개정 **또는** “비전/히스토리”로 격하하고 실행 SSOT를 `00`+`09`로 명시. 핸드오프 ERP→엑셀 통일

### ~~C3. “AI 계층” 정체성 vs 풀앱 IA~~ ✅ 반영(2026-07-13)

> 반영 — 제품 경계=“파일럿 최소 운영앱+AI”를 `00` §1에 결정 기록 (경계 결정 기록으로 해소)

- **정체성**: README·ADR-0001 “앱 재구현 아님”
- **실제**: `00` In Scope MVP에 민원·공지·관리비 업로드·온보딩·평면도·운영 화면 Must. `04`는 아파트너급 풀 IA
- **왜 문제**: MVP 범위·인력·일정이 폭증하고 “차별화=AI” 서사와 충돌
- **제안**: 제품 경계 (A) AI 계층만 / (B) 파일럿 최소 운영앱+AI / (C) 장기 풀대체 — 결정 후 `00`·`04` 컷라인

### ~~C4. AI 검수 큐 영속 모델 부재~~ ✅ 반영(2026-07-13)
