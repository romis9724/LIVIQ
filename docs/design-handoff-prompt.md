````text
당신은 LIVIQ의 시니어 프로덕트 디자이너입니다. 아래 브리핑을 바탕으로 고충실(high-fidelity) UI를 설계·제작하세요.

# 0. 제품 한 줄 정의
LIVIQ는 아파트 관리의 "AI 검색·응대·요약 계층"입니다. 입주민 앱·관리 웹을 다시 만드는 게 아니라,
기존 시스템·문서 위에서 (1) 입주민의 반복 문의를 출처와 함께 1차 응대하고 (2) 관리사무소의
문서 검색·공지 초안 작성을 돕는 도구입니다. "신뢰"가 제품의 핵심 가치입니다.

대상 사용자: 전 연령 입주민 + 실무 관리자(관리소장·직원·시설담당·입대의). 한국어 UI.

# 1. 디자인에 직결되는 절대 규칙 (어기면 안 됨)
- AI 답변 화면에는 **항상 "출처 카드"(문서명·조항·페이지·원문 링크)**를 함께 보여준다.
  근거가 없을 때는 답을 지어낸 화면을 그리지 말고, **"담당자 연결" 폴백 상태**를 그린다.
- 개인정보는 화면에서 **마스킹**한다. 예: 이름 `홍*동`, 연락처 `010-****-1234`, 동호수 부분 가림.
- 관리자 공지는 **자동발송 UI 금지**. 반드시 "키워드 → AI 초안 → 편집/검수 → 발송 확인 다이얼로그" 흐름.
- AI 응답 품질 관리를 위한 **"AI 검수 큐"** 화면이 존재한다(신뢰도 낮은 답변 검토·승인/반려).
- 관리비는 확정 관리비 데이터(관리자 업로드)를 "조회"하고, AI는 "왜 이만큼 나왔는지" **설명만** 한다(계산 X).
- AI 답변 말투는 단정하지 않는다("규정상 ~로 보입니다, 정확한 적용은 관리사무소 확인").

# 2. 디자인 방향 (무드)
- 콘셉트: "Trustworthy Utility" — 차분하고 명료하며 신뢰감 있는 라이트 테마.
- 화려함보다 **위계와 가독성** 우선. 위계는 균일한 카드 그리드가 아니라 scale 대비·여백 리듬으로.
- 다크모드는 기본값이 아니다(전 연령 가독성). 시스템 설정은 존중하되 라이트가 1차.
- AI를 "마법"이 아니라 **근거를 보여주는 도구**로 표현(출처 카드를 항상 전면에).
- 안티 템플릿: 기본 카드 그리드 남발/스톡 히어로/그라데이션 블롭/획일 라운드·그림자 금지.

# 4. 만들 화면 (우선순위 P0=먼저)

## 입주민 웹 (모바일 우선, 하단 탭 5개)
- [P0] AI 비서: 대화형. 사용자 질문 → 스트리밍 답변 → **출처 카드** → 상태 배지(답변됨/검토 필요/담당자 연결) → 👍/👎. 하단 추천 질문 칩(주차 등록/공사 시간/반려동물 규정).
- [P0] 홈: 우리 단지 요약(공지 N건·처리중 민원·이번달 관리비) + 바로가기(비서 질문·민원접수·관리비).
- [P0] 온보딩: 로그인(Google로 시작) / 가입(2단계 — ① 필수·선택 동의, ② 초대코드·성명·생년월일·동호 입력, **만 14세 미만은 차단 에러 상태**) / 승인 대기·거절(사유 + 재신청)·비활성 상태 화면.
- [P1] 공지: 목록(말머리·중요 배지) / 상세.
- [P1] 민원·하자: 접수(사진 첨부, AI 카테고리 추천 칩) / 내 민원 목록 / 상세(처리 타임라인 + 상태 Pill).
- [P1] 관리비: 이번달 금액 + 추이 그래프 + 항목 상세, 그리고 **"왜 올랐나요?" → AI 설명 카드(전월·평균 대비, 출처=확정 관리비 데이터(업로드) 시점 명시)**.
- [P1] 알림함: 인앱 알림 목록(공지·민원 상태·승인 알림, 읽음/안읽음 배지) — 웹푸시가 아닌 앱 내 알림 목록.
- [P2] 나: 내 활동(질의·민원·피드백 이력) / 알림·세대 설정 / 개인정보 동의 관리.

## 관리자 웹 (데스크톱 우선, 좌측 사이드바)
- [P0] AI 검수 큐: 카드별 (질문·AI답변·근거·신뢰도) → [승인]/[수정 후 승인]/[반려].
- [P0] 공지 초안 작성: 키워드 입력 폼 → AI 초안 → 편집기 검수 → [발송] 확인 다이얼로그.
- [P0] 가입 승인: 대기 목록(이름 마스킹·생일 `19**-**-**`·동호 + 명부 대조 배지 일치 ✓ / 불일치 ⚠) + [승인] / [거절(사유)] + 명부 엑셀 업로드 패널(diff 결과: 신규 N · 매칭 유지 N · 전출 후보 N — 전출 후보는 개별 비활성화).
- [P0] 관리비 업로드 마법사: ① 파일 + 대상월 → ② 검증(행·컬럼·사유 오류 표, 오류 있으면 진행 불가) → ③ 미리보기(합계·평균·전월 대비) → ④ 확정("해당 월 전체 교체" 경고 다이얼로그) + 부과 현황(월·세대 조회).
- [P1] 대시보드: 자동해결률·환각률·질의당 토큰비용·민원 현황 위젯(데이터 시각화).
- [P1] 민원 관리: 접수함(AI 분류·우선순위 배지) → 배정/처리. DataTable(정렬·필터).
- [P1] 문서 관리: 업로드 + 공개범위 설정(전체/입주민/관리자/입대의) + 색인 상태(pending/indexing/indexed/failed, 재색인).
- [P2] 시설 관리: 설비 상태 목록(정상/점검/장애/위험) + AI 도우미("가능 원인 후보"로 표기, 단정 X).
- [P2] 단지 관리 — 평면도 좌표 에디터: 마커 드래그로 위치 지정, 타입 배지(기본 / 세대 오버라이드). (스키마 검증용 저충실 프로토타입 선행)

# 5. 모든 화면에 상태 변형을 그릴 것
로딩(스켈레톤 / AI는 스트리밍) · 빈 상태(안내+행동 유도) · 오류(원인+재시도) · 권한 없음(명확한 안내) · 정상.
권한 없음을 빈 화면으로 두지 말 것.

# 6. 핵심 컴포넌트(디자인 시스템)
Button(size/variant/로딩·비활성), SurfaceCard, ChatPanel(스트리밍·자동스크롤), CitationCard(출처),
ConfidenceBadge(답변됨/검토 필요/담당자 연결), FeedbackButtons(👍/👎+사유), StatusPill(처리·설비 상태),
EmptyState, DataTable(정렬·필터·페이지네이션), FormField(라벨·에러·도움말), Toast/Dialog(위험 액션 확인),
FileDropzone(드래그&드롭·클릭 선택·진행률·오류), Stepper(업로드/가입 마법사 단계 표시).

# 7. 접근성 (WCAG 2.2 AA) — 필수
- 시맨틱 HTML(header/nav/main/section), 의미 없는 div 중첩 금지.
- 키보드 전체 조작 + 가시적 focus ring, 논리적 tab 순서, skip-link.
- 명도 대비 본문 ≥ 4.5:1. **색만으로 상태 전달 금지**(아이콘/텍스트 병기).
- 폼 label 연결 + 에러 aria-describedby, 입력 글자 16px+(자동확대 방지), 터치 타깃 ≥ 44px.
- 스트리밍 답변·토스트는 aria-live. prefers-reduced-motion 존중.

# 8. 반응형 / 모션
- 검증 브레이크포인트: 320·375·768·1024·1440·1920. 가로 스크롤 0. 표·그래프는 자체 overflow-x:auto.
- 애니메이션은 transform/opacity만(레이아웃 속성 애니 금지). 토큰의 duration/ease 사용.

# 9. 콘텐츠/카피 (실제 한국어, lorem ipsum 금지)
존댓말·간결·전문용어 풀어쓰기. 예시 카피를 그대로 활용:
- 비서 질문: "인테리어 공사 가능한 시간 알려줘"
- 비서 답변: "평일 09:00~18:00에 가능합니다. 주말·공휴일은 제한됩니다." + 출처 "관리규약 제32조(공사시간)·p.12 [원문]"
- 폴백: "정확한 확인이 필요해요. 관리사무소 담당자에게 연결할까요?"
- 관리비 설명: "이번 달은 전월 대비 12% 높습니다. 난방비가 ₩18,200 늘어난 영향이에요. (기준: 2026-05 확정 관리비 데이터(업로드))"
- 오류: "일시적인 오류예요. 잠시 후 다시 시도하거나 담당자에게 연결할게요."

# 10. 산출물 & 형식
- 형식: **자체 완결형 HTML/CSS 아티팩트**(인라인 CSS, 외부 폰트·이미지·CDN 의존 금지 — 시스템 폰트 스택 사용).
  화면 1개 = 아티팩트 1개 권장. 입주민은 모바일 프레임(약 390px), 관리자는 데스크톱(약 1280px)으로.
- 먼저 (a) 디자인 토큰을 적용한 파운데이션 + 핵심 컴포넌트 1개 아티팩트를 만들고,
  승인 후 (b) P0 화면, (c) P1 화면 순으로 진행. 각 화면은 정상 + 주요 상태 변형 포함.
- 각 아티팩트 상단에 화면명·역할(입주민/관리자)·뷰포트를 주석으로 표기.

# 11. 하지 말 것
- 다크모드를 기본으로 두기 / 출처 없는 AI 답변 화면 / 개인정보 평문 노출.
- 공지 자동발송(검수 단계 생략) UI / 균일 카드 템플릿·스톡 히어로 / lorem ipsum.
- 외부 폰트·이미지 CDN 의존(아티팩트에서 로드 실패함).

# 12. 참고
````
